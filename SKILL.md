# whitebox-testing

白盒测试专家助手，帮助你进行代码结构分析和白盒测试。

## 触发条件

当你需要以下帮助时使用此 Skill：

- "帮我做白盒测试"
- "设计白盒测试用例"
- "提高代码覆盖率"
- "测试某个函数的边界条件"
- "分析代码路径"
- "生成测试用例"

## 执行流程

### 步骤 1: 选择需要测试的模块

1. 读取源文件
2. 按文件大小列出各个模块

### 步骤 2: 选择编程语言

使用 AskUserQuestion 工具让用户选择编程语言：

```
请选择被测代码的编程语言：
1. Python
2. Java
3. JavaScript/TypeScript
4. C/C++
5. 其他
```

如果用户选择 Python、Java、JavaScript/TypeScript 或 C/C++，尝试加载对应的语言特定指南：

- Python: `references/python.md`
- Java: `references/java.md`
- JavaScript/TypeScript: `references/javascript.md`
- C/C++: `references/clang.md`

### 步骤 3: 选择覆盖标准

使用 AskUserQuestion 工具让用户选择覆盖标准：

```
请选择测试覆盖标准：
1. 语句覆盖 - 确保每个可执行语句至少执行一次（最低要求）
2. 判定路径覆盖 - 测试所有独立路径（更全面）
```

### 步骤 4: 代码分析

#### 如果选择语句覆盖：

1. 读取源文件
2. 识别所有可执行语句（排除注释、docstring、import语句等）
3. 生成语句覆盖率需求清单

#### 如果选择判定路径覆盖：

1. 读取用户指定的源文件
2. 识别函数/方法结构
3. 计算圈复杂度（McCabe复杂度）
4. 绘制流程图（使用ASCII图形）
5. 列举所有独立路径

**流程图示例**：

对于以下代码：

```python
def process(a, b):
    if a > 0:           # 判定1
        x = a + b
        if b > 0:       # 判定2
            return x * 2
        else:
            return x
    else:
        return b
```

流程图：

```
     ┌─────────────┐
     │   开始      │
     └──────┬──────┘
            │
     ┌──────▼──────┐
     │  a > 0 ?   │────┐
     └──────┬──────┘    │
        ┌───┴───┐       │
        │       │       │
   ┌────▼───┐   │       │
   │b > 0 ? │   │       │
   └────┬───┘   │       │
      ┌─┴─┐     │       │
      │   │     │       │
 ┌────▼───┐│     │       │
 │x*2返回 ││     │       │
 └────┬───┘│     │       │
      │    │     │       │
      └────┼─────┘       │
           │             │
     ┌─────▼─────┐       │
     │ b返回/x返回│       │
     └─────┬─────┘       │
           │             │
     ┌─────▼─────┐       │
     │   结束    │◄──────┘
     └───────────┘
```

独立路径：

- 路径1: 开始 → a>0真 → b>0真 → x\*2返回 → 结束
- 路径2: 开始 → a>0真 → b>0假 → x返回 → 结束
- 路径3: 开始 → a>0假 → b返回 → 结束

### 步骤 5: 生成测试用例

#### 语句覆盖：

为每个未覆盖语句设计测试用例，确保：

- 每个可执行语句都被执行
- 使用Mock模拟外部依赖

#### 判定路径覆盖：

为每条独立路径设计测试用例，包含：

- 测试目标描述
- 覆盖的独立路径
- 预期的测试数据

### 步骤 6: 运行测试

根据项目使用的编程语言和测试框架，自动执行生成的测试用例，并收集覆盖率数据。

Python 示例：

```bash
python -m pytest tests/ -v --cov=app --cov-report=term-missing
```

### 步骤 7: 分析覆盖率

1. 读取覆盖率报告
2. 根据未覆盖的语句/路径，分析原因
3. 判断是否达到100%

### 步骤 8: 迭代优化

如果未达到100%：

1. 分析未覆盖的原因
2. 设计新的测试用例覆盖缺失的路径/语句
3. 返回步骤5

### 步骤 9: 运行所有测试

1. 运行所有测试
2. 修复引起测试失败的源码
3. 重新测试之前失败的用例直到所有失败的测试通过
4. 运行所有测试，重复以上步骤直到全量测试全部通过

## 注意事项

1. 判定路径覆盖是中等难度的覆盖标准，比语句覆盖更全面
2. 圈复杂度高的函数需要更多测试用例
3. 循环路径只需要覆盖循环2次即可满足判定路径要求
4. 异常处理路径（try-except-finally）也需要覆盖
5. 不可达代码需要记录并报告

## 圈复杂度风险评估

| 复杂度 V(G) | 风险级别 | 测试建议           |
| ----------- | -------- | ------------------ |
| 1-5         | 低       | 正常测试流程       |
| 6-10        | 中       | 增加边界测试用例   |
| 11-20       | 高       | 重点测试，考虑重构 |
| 21+         | 极高     | 必须重构拆分函数   |

## 未覆盖代码原因分类

| 原因类型 | 说明                     | 处理方式     |
| -------- | ------------------------ | ------------ |
| 测试缺失 | 有路径但未设计测试用例   | 补充测试用例 |
| 条件限制 | 某些条件在当前场景不可达 | 文档化说明   |
| 死代码   | 永远不会执行的代码       | 考虑删除     |
| Mock限制 | 外部依赖被mock绕过       | 集成测试覆盖 |
| 异常场景 | 极端错误处理路径         | 评估是否必要 |
